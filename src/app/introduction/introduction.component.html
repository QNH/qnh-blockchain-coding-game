Welcome to the QNH Blockchain coding game

This is the QNH Coding game, where there is no hype, but only technology. To us, blockchain is nothing but a way of working and a niche way of solving problems in certain use cases. While playing along with the game, you'll learn and explore certain use cases in which you can use blockchain.

This game has four tasks, including this one. Doing one task unlocks the next one. In this task, you have to prepare your system to work with the Ethereum blockchain. If you have done the steps below, click the "Submit" button below.

First, you need to install NodeJS and npm from the NPM website. As you'll probably be running this web application from your own device, this should have succeeded. If you have not done so, also run npm install on the directory. This will install application dependencies, but again, since this is Angular, this should have been done before even starting the app.

Now it might be useful to start interacting with an Ethereum blockchain node. For this, we will use Go Ethereum (GEth) instances. You can download Geth from the Ethereum download page. Before heading right into using this tool, we'd suggest to not connect to the main network yet. The Ethereum mainnet costs real money to use, as we will cover later on. Instead, this game is going to focus on smaller, free networks. Unless you are doing this game on your own, you will be handed an address of the node you will be using in this game. You can fill this address in at the bottom of this page. You can test if your Geth instance is working, by using geth attach address, where address should be replaced by the network address (including ws:// and the port number). This will open a console on the ethereum node found and the address. You can best leave this console open, as we will be using it later on.

Next up, we will be working with Ethereum Smart Contracts. The development of smart contracts is a bit different to other backend applications. We will cover the differences when we reach them. But this requires a different array of tools.

In order to use smart contracts, we first need to write them. Ethereum smart contracts can be written in many languages, such as Solidity, a language similar to Javascript and Viper, a language similar to python. The end goal is to generate binary code which can be used by the Ethereum network. For the coding game, we will focus on writing in Solidity. Solidity code is compiled into binary code using solc. You can get solc by executing npm install --global solc. This will allow the solcjs command in Windows command prompt or Bash, which we will use later. More information regarding Solidity can be found on the Solidity ReadTheDocs.

After installing the Solidity Compiler, you should be settled and ready to go. So, fill in below what node you want to connect with. After that, fill in the your given private key. Finally, press submit and continue with the coding game.

*: I want/need to setup my own Ethereum node. Well, luckily, a Github user by the name of jastam has made a Docker image that can setup a local Ethereum node. For this, you need to get Docker from their website. After doing so, you can run the image with docker run jastam/geth, where it is advised to give it a convertional name, run it with a volume (so that the chain is persistent) and if you want to continuniously run it, use it with restart unless-stopped. The total command should sum up to docker run --name geth-client --volume geth-volume --restart unless-stopped jastam/geth.
